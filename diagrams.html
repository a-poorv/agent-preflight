Product Architecture

Claude Agent Compass

A decision-first pre-flight layer that guides users before they delegate to a Claude Agent — reducing silent failures, token waste, and low-confidence delegation.

Diagram 1

The Core Problem We Solve

Users aren't blocked by capability. They're blocked by decisions.

Without Agent Compass
Decision blind spots
  • Should I use simple prompting or Claude Agent?
  • What will happen if I delegate this task?
  • Will my constraints even be respected?
  • Silent failures — no warnings, no checkpoints
  • Token waste from unconstrained retries
→
With Agent Compass
Decision clarity before execution
  • Clear recommendation: agent vs manual, with reason
  • Detected constraints shown upfront
  • Skill opportunities only when genuinely useful
  • Checkpoints auto-managed by risk level
  • Proactive intervention before silent failure

✅ Execution complete — outcome delivered, learning profile updated
Diagram 3

The 4-Engine Model

Each engine handles a distinct concern — together they form the full pre-flight loop.

⚡
Engine 1
Intent Router
INPrompt text, task history
OUTRecommendation (agent / manual), confidence score, 1-line reason
🛡
Engine 2
Constraint + Skill Engine
INPrompt text, system limits, existing skill bank
OUTConstraint chips · Skill opportunity panel (only on boundary detection)
🔔
Engine 3
Intervention Engine
INStep risk, mission risk level, progress ratio
OUTIntervention banner · Auto-escalated checkpoints · Manual override gate
🧠
Engine 4
Learning Profile
INTask type, outcome (success / manual)
OUTAdaptive checkpoint strictness · Per-type success rate (localStorage)
